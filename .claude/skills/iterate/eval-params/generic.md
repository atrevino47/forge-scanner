# Generic Evaluation Parameters

Score each 1-10. Total max: 100.

| # | Parameter | What to evaluate |
|---|-----------|-----------------|
| 1 | **Readability** | Can someone understand this file in one pass? Clear structure? |
| 2 | **Simplicity** | Is this the simplest solution that works? No over-engineering? |
| 3 | **Correctness** | Does it do what it's supposed to? No bugs, no edge case gaps? |
| 4 | **Elegance** | Is the solution clean? Does it feel like it was written with care? |
| 5 | **Maintainability** | Can this be modified safely 6 months from now? Clear extension points? |
| 6 | **Consistency** | Does it follow the patterns established elsewhere in the project? |
| 7 | **Naming** | Are all identifiers self-documenting? No abbreviations without context? |
| 8 | **DRY** | No unnecessary repetition? Shared logic extracted? |
| 9 | **Documentation** | Comments where non-obvious? Not over-commented? Purpose clear? |
| 10 | **Completeness** | Does it handle the full scope? Nothing left as TODO or placeholder? |
