# Frontend Evaluation Parameters

Score each 1-10. Total max: 140.

| # | Parameter | What to evaluate |
|---|-----------|-----------------|
| 1 | **Visual Hierarchy** | Can the eye find the most important info in <2 seconds? Clear reading path? |
| 2 | **Information Density** | Right balance of content vs whitespace? Data presented efficiently? |
| 3 | **Premium Feel** | Does it feel like a $100K product? Material quality, depth, polish? |
| 4 | **Readability** | Text contrast, size, spacing. Can every label be read instantly? |
| 5 | **Navigation Clarity** | Can the user always tell where they are and what to do next? |
| 6 | **Color Usage** | Accent used surgically? Semantic colors consistent? Palette cohesive? |
| 7 | **Typography** | Fonts used purposefully? Hierarchy clear through type alone? |
| 8 | **Component Consistency** | Do cards, chips, buttons follow a unified system? |
| 9 | **Responsiveness** | Works at different widths? Layouts flexible? Touch targets adequate? |
| 10 | **Emotional Impact** | Does the design create the intended feeling? Does the CTA compel action? |
| 11 | **Micro-interactions** | Hover states, focus rings, transitions, cursor feedback present and polished? |
| 12 | **Spatial Rhythm** | Vertical cadence consistent? Padding/margin system coherent? Grid aligned? |
| 13 | **Data Storytelling** | Does the page tell a narrative? Does each section build toward the goal? |
| 14 | **Accessibility** | WCAG AA contrast? Focus indicators? Semantic HTML? ARIA labels? |
