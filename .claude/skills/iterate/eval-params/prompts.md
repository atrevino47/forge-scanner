# Prompt Evaluation Parameters

Score each 1-10. Total max: 140.

| # | Parameter | What to evaluate |
|---|-----------|-----------------|
| 1 | **Specificity** | Does the prompt give precise instructions, not vague guidance? |
| 2 | **Role Clarity** | Is the AI's role, personality, and constraints clearly defined? |
| 3 | **Output Format** | Is the expected output shape/format specified? JSON schema? Examples? |
| 4 | **Context Usage** | Does the prompt use available data effectively? No wasted context? |
| 5 | **Tone & Voice** | Consistent personality? Appropriate for the audience? |
| 6 | **Conversion Intent** | Does the prompt drive toward the business goal? Every instruction serves conversion? |
| 7 | **Guardrails** | Are there clear "never do this" rules? Hallucination prevention? |
| 8 | **Edge Cases** | How does the prompt handle missing data? Empty states? Unusual inputs? |
| 9 | **Adaptability** | Does the prompt work across different scenarios? Dynamic enough? |
| 10 | **Conciseness** | No redundant instructions? Every sentence earns its place? |
| 11 | **Example Quality** | Are examples provided? Do they demonstrate edge cases, not just happy paths? |
| 12 | **Token Efficiency** | Prompt fits within budget? Not bloated with unnecessary context? |
| 13 | **Safety** | No prompt injection vectors? Sensitive data handled? |
| 14 | **Measurability** | Can you tell if the prompt worked? Are success criteria implicit in the instructions? |
