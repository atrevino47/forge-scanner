# FIX-0011: Remove transform: none from visibility CSS injection

**Audit ref:** AUDIT-002, E2E browser test (FIX-0010 partial — breaks carousels)
**Assigned to:** Backend
**Priority:** P1 (blocks pipeline — carousel content disappears)
**Status:** pending

## Problem

FIX-0010 injected `transform: none !important` to force hidden elements visible. This works for scroll-triggered animations (which use `translateY` to hide elements) but BREAKS carousels/sliders. Carousels position items using `translateX()` — removing transforms causes all items to pile at position 0, and the container's `overflow: hidden` hides them. The result: carousel sections appear completely blank in screenshots (as seen on blotato.com between the hero and testimonials).

## File(s) to modify

- `src/lib/screenshots/client.ts` — the `page.addStyleTag` content injected after the scroll routine

## Required change

Remove `transform: none !important` from the injected CSS. Keep everything else.

**Before:**
```css
*, *::before, *::after {
  opacity: 1 !important;
  visibility: visible !important;
  clip-path: none !important;
  -webkit-clip-path: none !important;
  transform: none !important;
  transition: none !important;
  animation: none !important;
}
```

**After:**
```css
*, *::before, *::after {
  opacity: 1 !important;
  visibility: visible !important;
  clip-path: none !important;
  -webkit-clip-path: none !important;
  transition: none !important;
  animation: none !important;
}
```

Just remove the `transform: none !important;` line. That's it.

## Constraints

- Do NOT modify the scroll routine
- Do NOT modify any other files
