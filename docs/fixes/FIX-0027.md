# FIX-0027: Screenshot presentation — device chrome + loading states

**Audit ref:** AUDIT-003, Phase D.3
**Assigned to:** Frontend
**Priority:** P2 (breaks user flow)
**Status:** pending

## Problem
Screenshots render as raw images in boxes with no visual distinction between desktop and mobile captures. No device chrome, no size differentiation, no loading placeholder while images load. A user can't tell at a glance whether they're looking at a desktop or mobile screenshot. Failed image loads show nothing — no fallback.

## File(s) to modify
- `src/components/scan/ScreenshotCard.tsx` — device chrome, image loading, size differentiation
- `src/components/scan/StageSection.tsx` — adjust grid sizing for mobile vs desktop cards

## Required change
1. **Device badge**: Add a small pill badge below/on the image showing viewport type — laptop icon + "Desktop" or phone icon + "Mobile" using lucide-react icons. Position as an overlay in the top-right corner of the image container with semi-transparent dark background.
2. **Size differentiation**: Mobile screenshots render at ~60% width of desktop screenshots within the grid. In the 2-column grid, a mobile screenshot should be narrower with centered alignment, not stretched to fill the column.
3. **Image loading state**: Replace bare `<img>` with a loading-aware wrapper:
   - While loading: show a shimmer skeleton matching the approximate aspect ratio (16:10 for desktop, 9:16 for mobile)
   - On load: fade the image in (opacity 0→1, 0.3s)
   - On error: show a subtle error state with icon + "[COPY: screenshot unavailable]" text
4. **Image container aspect ratio**: Use `aspect-video` for desktop screenshots and `aspect-[9/16]` for mobile screenshots instead of the current max-height constraint.

## Constraints
- Do NOT modify Backend or AI Engine files
- Do NOT change the annotation dot positioning logic — dots must still render correctly on the image
- Annotation positions are stored as percentages — they should still work regardless of image container size
- Use lucide-react for device icons (Monitor, Smartphone)

## Verification
1. Desktop screenshots show laptop icon badge, mobile screenshots show phone icon badge
2. Mobile screenshots visually narrower than desktop in the grid
3. Images fade in on load (not instant appear)
4. Failed image loads show error fallback (not blank space)
5. `npx tsc --noEmit` passes
