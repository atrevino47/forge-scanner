# FIX-0004a: Make email optional in capture-info when socialConfirmation is provided

**Audit ref:** AUDIT-001, BUG-3
**Assigned to:** Backend
**Priority:** P2 (breaks user flow)
**Status:** pending

## Problem

The SocialConfirmation component sends a capture-info request with `email: ''` when the user confirms their social handle. The Zod schema requires `email` to be a non-empty valid email. This causes social confirmation to always return 400. This is the Backend half of a cross-agent fix — the contract type `CaptureInfoRequest` in `contracts/api.ts` will be updated by the Orchestrator to make `email` optional before this ticket is dispatched.

## File(s) to modify

- `src/app/api/scan/capture-info/route.ts` — lines 30-42: the `CaptureInfoSchema` Zod object

## Required change

Update the Zod schema so that `email` is optional. When `socialConfirmation` is present and `email` is not provided, the request should still be accepted. Add a `.refine()` to ensure that either `email` or `socialConfirmation` is present (at least one must be provided).

**Before:**
```typescript
const CaptureInfoSchema = z.object({
  scanId: z.string().min(1, 'Scan ID is required'),
  leadId: z.string().min(1, 'Lead ID is required'),
  email: z
    .string()
    .min(1, 'Email is required')
    .email('Must be a valid email address'),
  phone: z.string().optional(),
  fullName: z.string().optional(),
  businessName: z.string().optional(),
  providedSocials: ProvidedSocialsSchema,
  socialConfirmation: SocialConfirmationSchema,
});
```

**After:**
```typescript
const CaptureInfoSchema = z.object({
  scanId: z.string().min(1, 'Scan ID is required'),
  leadId: z.string().min(1, 'Lead ID is required'),
  email: z.string().email('Must be a valid email address').optional(),
  phone: z.string().optional(),
  fullName: z.string().optional(),
  businessName: z.string().optional(),
  providedSocials: ProvidedSocialsSchema,
  socialConfirmation: SocialConfirmationSchema,
}).refine(
  (data) => data.email || data.socialConfirmation,
  { message: 'Either email or socialConfirmation must be provided' }
);
```

Also update the handler logic: the `updateData` object (line 129) currently always includes `email`. Guard it:

**Before (line 129):**
```typescript
const updateData: Record<string, string> = {
  email,
  capture_method: 'direct',
```

**After:**
```typescript
const updateData: Record<string, string> = {
  capture_method: 'direct',
  ...(email && { email }),
```

## Constraints

- Do NOT modify any other route
- The updated schema must match the updated `CaptureInfoRequest` contract type (where `email` is now optional)
- Requests with `email` but no `socialConfirmation` must still work exactly as before
- Requests with `socialConfirmation` but no `email` must now succeed

## Cross-agent context

This is sub-ticket **a** (Backend). Apply this FIRST.

The Orchestrator will update `contracts/api.ts` to change `email: string` to `email?: string` in the `CaptureInfoRequest` interface before this ticket is dispatched.

Sub-ticket **b** (Frontend, FIX-0004b) will update `SocialConfirmation.tsx` to stop sending `email: ''`. It runs after this ticket.
