# FIX-0019: Pass booking source through Cal.com metadata

**Audit ref:** AUDIT-001, MISSING item 12
**Assigned to:** Frontend
**Priority:** P4 (wiring)
**Status:** pending

## Problem

The spec (§12) requires tracking which CTA triggered a booking: `banner_cta`, `ai_agent`, `results_cta`, etc. The Cal.com webhook handler (`/api/followup/webhook/calcom`) already reads `metadata.source` and maps it to a `BookingSource` type. But the Frontend's `CalcomContext` doesn't pass a `source` parameter when opening Cal.com — so every booking is logged as the default `banner_cta`.

## File(s) to modify

- `src/components/providers/CalcomContext.tsx` — add `source` to the context
- `src/components/shared/CalcomModal.tsx` — pass `source` as Cal.com metadata
- `src/components/shared/TopBanner.tsx` — pass `source: 'banner_cta'` when opening
- `src/components/scan/ScanLayout.tsx` — pass `source: 'results_cta'` for the blueprint CTA button

## Required change

### 1. CalcomContext.tsx — add source to state and openCalcom signature

```typescript
interface CalcomPrefill {
  name?: string;
  email?: string;
  phone?: string;
  source?: string; // NEW: booking source for tracking
}
```

Also add `source` to the context value so CalcomModal can read it:

```typescript
interface CalcomContextValue {
  isOpen: boolean;
  hasBooked: boolean;
  showConfirmation: boolean;
  prefill: CalcomPrefill;
  openCalcom: (prefill?: CalcomPrefill) => void;
  closeCalcom: () => void;
  onBookingSuccess: () => void;
}
```

No changes needed to the interface — `source` is already inside `CalcomPrefill` which is exposed via `prefill`.

### 2. CalcomModal.tsx — pass source as Cal.com metadata

In the `Cal` component, add metadata with the source. The `@calcom/embed-react` `Cal` component accepts a `config` prop with metadata:

After building the `calLink` string, add the source to the config:

```typescript
<Cal
  calLink={calLink}
  style={{ width: '100%', height: '100%', overflow: 'auto', minHeight: 480 }}
  config={{
    layout: 'month_view',
    theme: 'dark',
    metadata: {
      source: prefill.source ?? 'banner_cta',
    },
  }}
/>
```

### 3. TopBanner.tsx — pass source when opening

```typescript
onClick={() => openCalcom({ source: 'banner_cta' })}
```

### 4. ScanLayout.tsx — pass source for the results CTA

The "Book a Call" button after the blueprint (~line 439):

```typescript
onClick={() => openCalcom({
  email: state.capturedEmail ?? undefined,
  phone: state.capturedPhone ?? undefined,
  source: 'results_cta',
})}
```

Note: `handleOpenCalcom` at line 364 should also include the source. Update it:

```typescript
const handleOpenCalcom = (source: string = 'results_cta') => {
  openCalcom({
    email: state.capturedEmail ?? undefined,
    phone: state.capturedPhone ?? undefined,
    source,
  });
};
```

## Constraints

- Do NOT modify the Cal.com webhook handler (Backend) — it already reads `metadata.source`
- The `source` field must be optional in `CalcomPrefill` — existing callers without a source still work
- Default source is `banner_cta` (matches current behavior)
