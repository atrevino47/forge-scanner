# FIX-0014: Build middleware.ts for auth refresh and admin protection

**Audit ref:** AUDIT-001, MISSING-5
**Assigned to:** Backend
**Priority:** P3 (production reliability)
**Status:** pending

## Problem

No `src/middleware.ts` exists. The app needs middleware for two things:
1. **Supabase auth session refresh** — the `@supabase/ssr` package requires middleware to refresh expired auth tokens on each request
2. **Admin route protection** — `/admin` and `/admin/scan/[id]` should be restricted to authenticated users with team/admin roles

For now, since admin pages are stubs and auth is optional (per spec), a minimal middleware that only handles Supabase session refresh is sufficient. Admin protection can be added when the admin pages are built.

## File(s) to modify

- `src/middleware.ts` — **new file**

## Required change

Create `src/middleware.ts` with Supabase auth session refresh using the `@supabase/ssr` pattern for Next.js App Router.

```typescript
import { createServerClient } from '@supabase/ssr';
import { NextResponse, type NextRequest } from 'next/server';

export async function middleware(request: NextRequest) {
  let supabaseResponse = NextResponse.next({ request });

  const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL;
  const supabaseAnonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY;

  if (!supabaseUrl || !supabaseAnonKey) {
    return supabaseResponse;
  }

  const supabase = createServerClient(supabaseUrl, supabaseAnonKey, {
    cookies: {
      getAll() {
        return request.cookies.getAll();
      },
      setAll(cookiesToSet) {
        for (const { name, value } of cookiesToSet) {
          request.cookies.set(name, value);
        }
        supabaseResponse = NextResponse.next({ request });
        for (const { name, value, options } of cookiesToSet) {
          supabaseResponse.cookies.set(name, value, options);
        }
      },
    },
  });

  // Refresh the auth session if it exists
  await supabase.auth.getUser();

  return supabaseResponse;
}

export const config = {
  matcher: [
    // Match all routes except static files and API health check
    '/((?!_next/static|_next/image|favicon.ico|api/health).*)',
  ],
};
```

## Constraints

- Do NOT add admin route protection yet — admin pages are stubs
- Do NOT redirect unauthenticated users — auth is optional per spec
- The middleware must only refresh the session, not block any routes
- Use `@supabase/ssr` which is already installed
