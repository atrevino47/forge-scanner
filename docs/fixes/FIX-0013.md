# FIX-0013: Use waitUntil for pipeline so it survives on Vercel

**Audit ref:** AUDIT-001, RUNTIME-1
**Assigned to:** Backend
**Priority:** P3 (production reliability)
**Status:** pending

## Problem

`runScreenshotPipeline()` is called fire-and-forget with `.catch()` only (line 149-158). The POST handler returns the response immediately. On Vercel serverless, the function may be killed before the 13-step pipeline completes. Works in `npm run dev` but will fail in production.

Next.js 15+ provides `waitUntil()` via the `after()` API from `next/server` to keep the serverless function alive after the response is sent.

## File(s) to modify

- `src/app/api/scan/start/route.ts` — lines 147-158: the pipeline dispatch block

## Required change

Use Next.js `after()` to run the pipeline after the response is sent. Import `after` from `next/server`.

**Before:**
```typescript
import { NextRequest, NextResponse } from 'next/server';
```
```typescript
// (g) Dispatch the screenshot pipeline as a background job
// Fire and forget — pipeline updates DB as it goes, SSE picks up changes
runScreenshotPipeline({
  scanId: scan.id,
  leadId: lead.id,
  websiteUrl: normalizedUrl,
}).catch((err: unknown) => {
  console.error(
    `[scan/start] Pipeline failed for scan ${scan.id}:`,
    err
  );
});
```

**After:**
```typescript
import { NextRequest, NextResponse, after } from 'next/server';
```
```typescript
// (g) Dispatch the screenshot pipeline — runs after response is sent
// Uses Next.js after() to keep the serverless function alive on Vercel
after(
  runScreenshotPipeline({
    scanId: scan.id,
    leadId: lead.id,
    websiteUrl: normalizedUrl,
  }).catch((err: unknown) => {
    console.error(
      `[scan/start] Pipeline failed for scan ${scan.id}:`,
      err
    );
  })
);
```

## Constraints

- Do NOT modify any other part of the route handler
- Do NOT change the pipeline function itself
- The `.catch()` must remain — `after()` does not handle rejections
- `after()` is available in Next.js 15+ from `next/server`

## Verification

The Orchestrator will verify by:
1. `npx tsc --noEmit` — zero errors
2. Confirm `after` is imported from `next/server`
3. Confirm `runScreenshotPipeline` is wrapped in `after()`
4. Confirm `.catch()` is still present inside the `after()` call
