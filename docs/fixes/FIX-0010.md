# FIX-0010: Force all elements visible before screenshot capture

**Audit ref:** AUDIT-002, E2E browser test (FIX-0008 insufficient)
**Assigned to:** Backend
**Supersedes:** FIX-0008 (scroll approach alone doesn't work)
**Priority:** P1 (blocks pipeline — screenshots still have blank spaces)
**Status:** pending

## Problem

FIX-0008 added a scroll routine to trigger scroll-based animations, but blank spaces persist. The root cause is that modern sites use IntersectionObserver, CSS initial states (`opacity: 0`, `transform: translateY(50px)`, `clip-path: inset(100%)`), and JS animation libraries that set elements as invisible until triggered. Programmatic `window.scrollBy` inside `page.evaluate` doesn't reliably fire IntersectionObserver callbacks in headless browsers.

The fix is to inject CSS that forcefully overrides all animation-related hiding styles, making every element visible regardless of animation state, THEN capture the screenshot.

## File(s) to modify

- `src/lib/screenshots/client.ts` — the `capturePageWithMetadata` function, between the scroll routine and the screenshot capture

## Required change

After the scroll routine (line 176) and before the screenshot capture (line 178), inject a `<style>` tag that forces all elements to be visible. This must happen AFTER scrolling (so any JS-based content loading has a chance to fire) but BEFORE the screenshot.

Add this block between lines 176 and 178:

```typescript
// Force all elements visible — override animation initial states
// (opacity: 0, transform, clip-path, visibility hidden, etc.)
try {
  await page.addStyleTag({
    content: `
      *, *::before, *::after {
        opacity: 1 !important;
        visibility: visible !important;
        clip-path: none !important;
        -webkit-clip-path: none !important;
        transform: none !important;
        transition: none !important;
        animation: none !important;
      }
    `,
  });
  // Brief wait for reflow after style injection
  await page.waitForTimeout(500);
} catch (styleError: unknown) {
  const styleMessage =
    styleError instanceof Error ? styleError.message : String(styleError);
  console.warn(
    \`[screenshots/client] Style injection failed for \${url}: \${styleMessage}. Capturing as-is.\`,
  );
}
```

**Important:** This uses `!important` to override any inline styles or specificity-heavy rules that animation libraries set. The `transform: none` reset means positioned elements will snap to their final layout position. The `transition: none` and `animation: none` prevent any visual artifacts during capture.

## Constraints

- Do NOT remove the existing scroll routine — keep it. The scroll still helps trigger lazy-loaded content and IntersectionObserver callbacks that load images/iframes. The style injection is a second layer of defense.
- Do NOT apply this style tag before scrolling — the scroll may trigger content loading (lazy images, dynamic sections). The style override should come after.
- The style injection must be wrapped in try/catch — if it fails, capture whatever is visible.
- Do NOT modify any other files.
