# FIX-0025: Google OAuth soft prompt — "Save your results" UI

**Audit ref:** AUDIT-003, Phase A item 6
**Assigned to:** Frontend
**Priority:** P4 (wiring)
**Status:** pending

## Problem
The spec calls for an optional Google OAuth soft prompt after the blueprint is generated — a "Save your results" nudge that lets users create an account to revisit their scan later. The backend infrastructure exists (auth callback route at `src/app/api/auth/callback/route.ts`, auth utilities in `src/lib/auth/config.ts`, middleware session refresh in `src/middleware.ts`), but there is ZERO frontend UI: no OAuth button, no auth context/provider, no Supabase browser client initialization, and no prompt component.

## File(s) to modify
- `src/components/scan/SaveResultsPrompt.tsx` (new) — the OAuth soft prompt component
- `src/components/providers/SupabaseProvider.tsx` (new) — Supabase browser client + auth context
- `src/app/layout.tsx` — wrap app with SupabaseProvider
- `src/components/scan/ScanLayout.tsx` — render SaveResultsPrompt after blueprint section

## Required change
1. **SupabaseProvider:** Create a React context that initializes `createBrowserClient()` from `@supabase/ssr` and exposes `{ supabase, user, signInWithGoogle() }` to descendants. Include a `useAuth()` hook.
2. **SaveResultsPrompt component:**
   - Renders after blueprint is generated (conditional on `state.blueprintReady && !user`)
   - Glassmorphism card with GSAP fadeSlideUp entrance
   - Google icon + "[COPY: Save your results]" button
   - Clicking triggers `supabase.auth.signInWithOAuth({ provider: 'google', options: { redirectTo: currentUrl } })`
   - After sign-in redirect, component hides (user is now authenticated)
   - Dismissible with an "x" or "[COPY: Skip]" link
   - Non-blocking — does NOT gate any content
3. **Layout integration:** Add SupabaseProvider to the root layout provider stack
4. **ScanLayout integration:** Render `<SaveResultsPrompt />` below the BlueprintCTA section, only when blueprint is ready and user is not authenticated

## Constraints
- Do NOT modify Backend files — auth callback route already works
- Do NOT modify `contracts/` — no contract changes needed
- OAuth is explicitly "optional, not a gate" — this MUST be non-blocking
- Use GSAP for entrance animation (not Framer Motion)
- Follow glassmorphism design system from design tokens

## Verification
The Orchestrator will verify by:
1. Confirming SaveResultsPrompt.tsx exists and renders conditionally
2. Confirming SupabaseProvider.tsx initializes browser client correctly
3. Confirming OAuth is non-blocking (no content gated behind sign-in)
4. Confirming GSAP animation exists (not CSS transition)
5. `npx tsc --noEmit` passes
