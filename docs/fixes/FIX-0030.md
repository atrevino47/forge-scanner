# FIX-0030: Chat design polish — message animations + UX

**Audit ref:** AUDIT-003, Phase D.5
**Assigned to:** Frontend
**Priority:** P2 (breaks user flow)
**Status:** pending

## Problem
The chat panel feels functional but not premium. Messages appear instantly with no entrance animation. The streaming cursor is invisible (blink keyframes missing — fixed in FIX-0026). The input doesn't auto-focus when the panel opens. The new-message badge on ChatToggle doesn't pulse. The typing indicator entrance/exit is abrupt. These gaps make the AI Sales Agent feel robotic instead of conversational.

## File(s) to modify
- `src/components/chat/ChatContainer.tsx` — auto-focus, message tracking for new-message badge
- `src/components/chat/ChatMessage.tsx` — entrance animation for non-streaming messages
- `src/components/chat/ChatInput.tsx` — auto-focus on panel open, submit button feedback
- `src/components/chat/ChatToggle.tsx` — new message badge pulse animation
- `src/components/chat/TypingIndicator.tsx` — softer scale values, entrance/exit animation

## Required change
1. **Message entrance animation**: In ChatMessage, wrap non-streaming messages in a GSAP fadeSlideUp on mount (y: 12→0, opacity: 0→1, 0.3s, power2.out). Streaming messages skip this (they build character by character).
2. **Auto-focus**: In ChatContainer, when `isOpen` transitions from false→true, focus the textarea input after a 100ms delay (let the panel animation complete first). Use a ref forwarded from ChatInput.
3. **Smooth auto-scroll**: Replace instant `scrollTop =` with `gsap.to(container, { scrollTop: container.scrollHeight, duration: 0.3, ease: 'power2.out' })`.
4. **Submit button feedback**: When a message is being sent (streaming in progress), show a subtle loading spinner icon on the send button instead of just opacity change. Use the existing Loader2 icon with spin animation.
5. **New message badge pulse**: In ChatToggle, when `hasNewMessage` is true, add a GSAP pulse animation on the red dot badge (scale 1→1.3→1, repeat 3 times, 0.6s each). Also ensure ChatContainer sets `hasNewMessage` when a message arrives while panel is closed.
6. **Typing indicator polish**: In TypingIndicator, reduce scale range from 1.0→1.5 to 1.0→1.2. Adjust opacity from 1.0→0.3 to 1.0→0.5. Add fadeSlideUp entrance (y: 8→0, 0.2s) on mount.
7. **Streaming message visual distinction**: Give streaming (in-progress) assistant messages a slightly lower opacity (0.85) that transitions to 1.0 when message_complete fires.

## Constraints
- Do NOT modify Backend or AI Engine files
- Do NOT change the SSE event handling or message data format
- All animations use GSAP (not Framer Motion or CSS transitions)
- Do NOT break the existing data_card and calcom_embed metadata rendering
- Document animation sequences in comment blocks

## Verification
1. Non-streaming messages slide in with fadeSlideUp
2. Textarea auto-focuses when chat panel opens
3. Auto-scroll is smooth (not instant jump)
4. ChatToggle badge pulses when new message arrives with panel closed
5. `npx tsc --noEmit` passes
