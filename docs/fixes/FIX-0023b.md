# FIX-0023b: Data cards in chat — Backend stream parser

**Audit ref:** AUDIT-003, Phase A item 4
**Assigned to:** Backend
**Priority:** P3 (reliability)
**Status:** pending

## Problem
The chat stream API (`/api/chat/stream/[convId]/route.ts:179-275`) receives streaming tokens from Claude via `streamWithSonnet()` but has no logic to detect `[DATA_CARD:screenshotId]` markers in the AI's response or emit `data_card` SSE events to the frontend. The tokens pass through as plain text.

## File(s) to modify
- `src/app/api/chat/stream/[convId]/route.ts` — around line 179-275: add marker detection and SSE emission logic

## Required change
1. **Buffer incoming tokens** to detect the `[DATA_CARD:{screenshotId}]` marker pattern across chunk boundaries. Use a small rolling buffer (e.g., accumulate tokens until a complete marker is detected or ruled out).
2. **When a marker is detected:**
   - Extract the `screenshotId` from the marker
   - Emit a `data_card` SSE event: `data: {"type":"data_card","screenshotId":"...","annotations":[...]}\n\n`
   - Fetch the annotations for that screenshotId from the database to include in the event
   - Strip the marker text from the displayed assistant message (don't show `[DATA_CARD:...]` to the user)
3. **When no marker:** pass tokens through as normal `content_delta` events
4. Handle edge cases: marker split across chunks, multiple markers in one response, invalid screenshotId (skip silently)

## Constraints
- Do NOT modify AI Engine files — FIX-0023a handles the system prompt
- Do NOT modify Frontend files — ChatContainer already handles `data_card` events
- Do NOT modify `contracts/events.ts` — the event type is already correct
- The emitted event must match the `data_card` shape in `contracts/events.ts:101`
- Marker parsing must not break or delay normal text streaming

## Verification
The Orchestrator will verify by:
1. Reading the stream route and confirming marker detection regex exists
2. Confirming `data_card` SSE event is emitted with correct shape
3. Confirming marker text is stripped from the displayed message
4. `npx tsc --noEmit` passes

## Cross-agent context
FIX-0023a (AI Engine) defines the `[DATA_CARD:{screenshotId}]` marker protocol in the system prompt. **FIX-0023a runs first.** This ticket parses those markers from the stream and converts them to SSE events.
