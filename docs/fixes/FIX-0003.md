# FIX-0003: Cal.com embed uses wrong calLink path

**Audit ref:** AUDIT-001, BUG-4
**Assigned to:** Frontend
**Priority:** P2 (breaks user flow)
**Status:** pending

## Problem

The Cal.com modal hardcodes `calLink` to `forge-digital/strategy-call` (line 114), which doesn't match the real Cal.com account. The env var `NEXT_PUBLIC_CALCOM_EMBED_URL` is set to `https://cal.com/adrian-trevino-kjqlqe/30min`. The `@calcom/embed-react` `Cal` component expects a `calLink` in `username/event-type` format. The modal currently loads a 404 or wrong event — booking doesn't work.

## File(s) to modify

- `src/components/shared/CalcomModal.tsx` — line 114: hardcoded `calLink` value

## Required change

Read the Cal.com path from the `NEXT_PUBLIC_CALCOM_EMBED_URL` environment variable and strip the `https://cal.com/` prefix. The pre-fill query string logic should remain as-is.

**Before:**
```typescript
const calLink = `forge-digital/strategy-call${qs ? `?${qs}` : ''}`;
```

**After:**
```typescript
const calcomUrl = process.env.NEXT_PUBLIC_CALCOM_EMBED_URL ?? '';
const calPath = calcomUrl.replace(/^https?:\/\/cal\.com\//, '');
const calLink = `${calPath}${qs ? `?${qs}` : ''}`;
```

## Constraints

- Do NOT modify any other component or file
- Do NOT change the prefill logic (lines 109-113)
- The `NEXT_PUBLIC_CALCOM_EMBED_URL` env var is already set — do not add a fallback to the old hardcoded value
- This is a `'use client'` component, so `process.env.NEXT_PUBLIC_*` vars are available at build time

## Verification

The Orchestrator will verify by:
1. `npx tsc --noEmit` — zero errors
2. Confirm the hardcoded `forge-digital/strategy-call` string is gone
3. Confirm the `calLink` is derived from `NEXT_PUBLIC_CALCOM_EMBED_URL`
4. Trace: env var `https://cal.com/adrian-trevino-kjqlqe/30min` → strip prefix → `adrian-trevino-kjqlqe/30min` → append query string if present
