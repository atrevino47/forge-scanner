# FIX-0012: Simplify CSS injection to only opacity + visibility

**Audit ref:** AUDIT-002, E2E browser test (FIX-0010/0011 too aggressive)
**Assigned to:** Backend
**Priority:** P1 (blocks pipeline — carousels still blank)
**Status:** pending

## Problem

The CSS injection from FIX-0010 includes `transition: none !important`, `animation: none !important`, and `clip-path: none !important`. These kill carousel initialization — carousel libraries use CSS transitions and animations to position items on load. The result: carousel sections remain blank.

The fix: strip the injection down to ONLY what fixes the actual problem (scroll-triggered hidden content): `opacity` and `visibility`. Let everything else run naturally. Also, move the style injection to happen AFTER a longer wait so carousels have time to initialize via JS before we force visibility.

## File(s) to modify

- `src/lib/screenshots/client.ts` — the `page.addStyleTag` block (lines ~178-200)

## Required change

Replace the current style injection block with a simpler version. Also restructure the timing: wait for JS carousels/animations to initialize FIRST, then inject the minimal CSS override, then capture.

**Replace the entire style injection block (lines 178-200) with:**

```typescript
// Wait for JS-driven content (carousels, lazy-loaded sections) to initialize
await page.waitForTimeout(2000);

// Force hidden animated elements visible — minimal override
// ONLY opacity and visibility. Do NOT override transform, animation,
// transition, or clip-path — those break carousels and positioned elements.
try {
  await page.addStyleTag({
    content: `
      *, *::before, *::after {
        opacity: 1 !important;
        visibility: visible !important;
      }
    `,
  });
  await page.waitForTimeout(300);
} catch (styleError: unknown) {
  const styleMessage =
    styleError instanceof Error ? styleError.message : String(styleError);
  console.warn(
    \`[screenshots/client] Style injection failed for \${url}: \${styleMessage}. Capturing as-is.\`,
  );
}
```

Note the changes:
1. Added `await page.waitForTimeout(2000)` BEFORE style injection (gives carousels time to init)
2. CSS is now ONLY `opacity: 1` and `visibility: visible` — nothing else
3. Reduced post-injection wait from 500ms to 300ms (less needed since we're doing less)
4. Updated the comment to explain WHY we don't override other properties
