# FIX-0023a: Data cards in chat — AI Engine system prompt protocol

**Audit ref:** AUDIT-003, Phase A item 4
**Assigned to:** AI Engine
**Priority:** P3 (reliability)
**Status:** pending

## Problem
The `data_card` SSE event is defined in `contracts/events.ts:101`, the frontend handles it in `ChatContainer.tsx:144-157`, and `DataCard.tsx` renders it. But the AI Sales Agent never triggers data card emission because: (1) the system prompt (`sales-agent-system.ts:140-148`) only vaguely mentions "the frontend may display inline data cards" without instructing the AI HOW to trigger them, and (2) no markup protocol exists for the AI to signal "show this screenshot."

## File(s) to modify
- `src/lib/prompts/sales-agent-system.ts` — around line 140-148: add explicit protocol for data card emission

## Required change
1. Add a `## DATA CARD PROTOCOL` section to the Sales Agent system prompt that instructs the AI to use a special inline marker when referencing a specific scan finding:
   - Format: `[DATA_CARD:screenshotId]` — where screenshotId is the UUID of the screenshot
   - The system prompt must explain: "When you reference a specific finding from the scan, include the marker `[DATA_CARD:{screenshotId}]` in your response. The frontend will render an interactive card showing the annotated screenshot inline in the chat."
2. Include the available screenshotIds in the system prompt context (they're already passed as scan data — ensure the prompt surfaces them as referenceable IDs)
3. Instruct the AI to use data cards strategically: in the opener message (reference the most critical finding), when discussing specific issues, and when building urgency

## Constraints
- Do NOT modify Frontend files — ChatContainer and DataCard already handle the event
- Do NOT modify `contracts/events.ts` — the event type is already correct
- Do NOT modify Backend stream API — FIX-0023b handles that
- The marker format must be parseable by regex in the stream API

## Verification
The Orchestrator will verify by:
1. Reading sales-agent-system.ts and confirming DATA_CARD protocol section exists
2. Confirming the marker format is documented: `[DATA_CARD:{screenshotId}]`
3. Confirming screenshotIds are referenced in the prompt context
4. `npx tsc --noEmit` passes

## Cross-agent context
FIX-0023b (Backend) will parse the AI's `[DATA_CARD:...]` markers from the streaming response, strip them from the displayed text, and emit `data_card` SSE events. **This ticket (FIX-0023a) runs first** — the protocol must be defined before the parser can be built.
