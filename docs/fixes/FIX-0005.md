# FIX-0005: Guard CapturePrompt visibility on leadId

**Audit ref:** AUDIT-001, RUNTIME-4
**Assigned to:** Frontend
**Priority:** P2 (breaks user flow)
**Status:** pending

## Problem

SSE sends `capture_prompt` after 15 seconds. The `leadId` comes from the initial data fetch (`/api/scan/results/{scanId}`). If that fetch is slower than 15 seconds, `leadId` is still null when the CapturePrompt renders. The CapturePrompt then sends `leadId ?? ''` which fails the Backend's Zod `min(1)` validation. This is a race condition — the capture form appears but submission silently fails.

## File(s) to modify

- `src/components/scan/ScanLayout.tsx` — line 469: the CapturePrompt visibility condition

## Required change

Add `state.leadId !== null` to the CapturePrompt rendering condition.

**Before:**
```tsx
{state.showCapturePrompt && (
  <CapturePrompt
```

**After:**
```tsx
{state.showCapturePrompt && state.leadId && (
  <CapturePrompt
```

## Constraints

- Do NOT modify any other condition or component
- Do NOT change the CapturePrompt component itself
- The `state.showCapturePrompt` flag must still be respected — this just adds a second guard

## Verification

The Orchestrator will verify by:
1. `npx tsc --noEmit` — zero errors
2. Confirm line 469 now has both `state.showCapturePrompt` and `state.leadId` in the condition
3. Trace: if leadId is null, CapturePrompt does not render even if showCapturePrompt is true
