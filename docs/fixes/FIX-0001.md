# FIX-0001: Screenshot IDs must be valid UUIDs

**Audit ref:** AUDIT-001, BUG-1
**Assigned to:** Backend
**Priority:** P1 (blocks pipeline)
**Status:** pending

## Problem

`generateScreenshotId()` produces IDs like `ss_lxyz123_abc12345` (custom text format). The `screenshots` table defines `id uuid primary key`. PostgreSQL rejects non-UUID values on insert, causing ALL screenshot DB inserts to fail silently. No screenshots are ever stored. The entire scan pipeline produces no visible output.

## File(s) to modify

- `src/lib/screenshots/pipeline.ts` — lines 582-587: `generateScreenshotId()` returns a custom string instead of a UUID

## Required change

Replace the body of `generateScreenshotId()` with `crypto.randomUUID()`.

**Before:**
```typescript
function generateScreenshotId(): string {
  const timestamp = Date.now().toString(36);
  const randomPart = Math.random().toString(36).substring(2, 10);
  return `ss_${timestamp}_${randomPart}`;
}
```

**After:**
```typescript
function generateScreenshotId(): string {
  return crypto.randomUUID();
}
```

## Constraints

- Do NOT modify any other function in this file
- Do NOT add imports — `crypto.randomUUID()` is available globally in Node.js 19+ and Next.js 15
- The return type must remain `string`

## Verification

The Orchestrator will verify by:
1. `npx tsc --noEmit` — zero errors
2. Confirm the function body is exactly `return crypto.randomUUID();`
3. Confirm the returned value matches UUID v4 format (`/^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i`)
