# FIX-0020: Email gate — smooth blur/unblur animation

**Audit ref:** AUDIT-003, Phase A item 1
**Assigned to:** Frontend
**Priority:** P2 (breaks user flow)
**Status:** pending

## Problem
When scan results load without email captured, `ScanLayout.tsx:446` applies `blur-sm` as an instant CSS class toggle. When the user submits their email, the blur removes instantly with no transition. The "Unlock Results" overlay (lines 496-512) also appears/disappears without animation. This feels abrupt and cheap — not the $100K premium experience the spec demands.

## File(s) to modify
- `src/components/scan/ScanLayout.tsx` — line 446: blur class toggle; lines 496-512: gate overlay

## Required change
1. Replace the instant `blur-sm` class toggle with a GSAP-driven blur animation:
   - **Blur entrance:** When `needsEmailGate` becomes true, animate `filter: blur(0px) → blur(8px)` over 0.6s with `power2.out` easing
   - **Unblur reveal:** When `emailCaptured` flips to true, animate `filter: blur(8px) → blur(0px)` over 0.8s with `power2.inOut` easing
   - Coordinate with a subtle scale: `scale(0.98) → scale(1)` during unblur for a "coming into focus" feel
2. Animate the email gate overlay:
   - **Entrance:** `fadeSlideUp` from bottom (translateY 20px → 0, opacity 0 → 1) over 0.5s
   - **Exit:** fade out over 0.3s before the unblur starts
3. Use GSAP timeline to sequence: overlay fades out (0.3s) → blur removes (0.8s)
4. Document the animation sequence in a comment block per CLAUDE.md rules

## Constraints
- Do NOT modify any files outside Frontend-owned directories
- Use GSAP presets from `src/lib/gsap-presets.ts` where applicable
- Do NOT use Framer Motion or CSS transitions for the blur effect
- The blur must still work as a functional gate (pointer-events-none, select-none while blurred)

## Verification
The Orchestrator will verify by:
1. Reading modified ScanLayout.tsx and confirming GSAP timeline exists for blur/unblur
2. Confirming `blur-sm` Tailwind class is replaced with GSAP-driven filter animation
3. Confirming animation sequence comment block exists
4. `npx tsc --noEmit` passes
