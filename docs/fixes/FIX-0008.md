# FIX-0008: Screenshots miss animated content and mobile captures are distorted

**Audit ref:** AUDIT-002, E2E browser test
**Assigned to:** Backend
**Priority:** P1 (blocks pipeline ‚Äî screenshots are the core product)
**Status:** pending

## Problem

Two related screenshot capture issues:

1. **Blank spaces in screenshots:** Sites using GSAP, Framer Motion, or scroll-triggered animations only render content when scrolled into view. The capture function uses `waitUntil: 'networkidle'` but never scrolls the page. The resulting screenshots have large blank areas where animated content should be ‚Äî the AI then has nothing to annotate in those regions.

2. **Mobile screenshots are stretched/distorted:** The mobile capture uses viewport 375x812 with `fullPage: true`, producing an extremely tall narrow image (375px wide, thousands of pixels tall). When displayed at card width in the results page, the image looks unnaturally stretched and doesn't represent how the site actually looks on a phone.

## File(s) to modify

- `src/lib/screenshots/client.ts` ‚Äî the `capturePageWithMetadata` function (line 108+) and `captureScreenshot` function (line 72+)

## Required change

### Change 1: Scroll page before capture to trigger animations

After the `page.goto()` call completes (line 130-149) and before the screenshot capture (line 152), add a scroll routine that:

1. Scrolls to the bottom of the page incrementally (not instant ‚Äî use `page.evaluate` with `window.scrollBy` in steps)
2. Scrolls back to the top
3. Waits 1-2 seconds for animations to settle

Example implementation:

```typescript
// After page.goto() and before captureScreenshot()

// Scroll down incrementally to trigger scroll-based animations
await page.evaluate(async () => {
  const scrollStep = Math.max(window.innerHeight * 0.8, 400);
  const maxScroll = document.body.scrollHeight;
  let currentScroll = 0;

  while (currentScroll < maxScroll) {
    window.scrollBy(0, scrollStep);
    currentScroll += scrollStep;
    await new Promise((r) => setTimeout(r, 200));
  }

  // Scroll back to top
  window.scrollTo(0, 0);
});

// Wait for animations to settle after scrolling
await page.waitForTimeout(1500);
```

### Change 2: Cap mobile screenshots to viewport height

In the `captureScreenshot` function or in `capturePageWithMetadata`, when `viewport === 'mobile'`:
- Use `fullPage: false` instead of `fullPage: true`
- This captures only the visible viewport (above-the-fold), which is what users actually see on their phone

```typescript
// In capturePageWithMetadata, when calling captureScreenshot:
const screenshotBuffer = await captureScreenshot(page, {
  fullPage: viewport === 'desktop', // full page for desktop, viewport-only for mobile
  viewport: viewportSize,
});
```

## Constraints

- Do NOT modify the viewport dimensions (375x812 mobile, 1440x900 desktop)
- Do NOT change the `waitUntil: 'networkidle'` strategy ‚Äî the scroll is an addition, not a replacement
- Do NOT modify any other files (pipeline.ts, social-detector.ts)
- Keep the navigation timeout handling as-is
- The scroll routine must be wrapped in try/catch ‚Äî if scrolling fails, still capture whatever loaded
