# FIX-0032: Results page progressive reveal + visual rhythm

**Audit ref:** AUDIT-003, Phase D.2
**Assigned to:** Frontend
**Priority:** P2 (breaks user flow)
**Status:** pending

## Problem
The results page loads all stage sections at once after the email gate unblurs. There's no progressive reveal, no visual rhythm between stages, and no sense of journey. All 5 stages + health summary + blueprint CTA appear simultaneously, creating an overwhelming wall of content. The user needs to be guided through the results stage by stage, building toward the CTA.

## File(s) to modify
- `src/components/scan/StageSection.tsx` — enhance scroll-triggered reveal with better stagger
- `src/components/scan/ScanLayout.tsx` — add visual separators between stages, enhance stage ordering presentation
- `src/components/scan/FunnelHealthSummary.tsx` — ensure scroll trigger only fires when in viewport

## Required change
1. **Stage entrance sequencing**: Each StageSection should have its own ScrollTrigger that fires when it enters 80% of the viewport. The section starts at opacity 0 + translateY 40px, then reveals with a coordinated sequence:
   - Beat 1: Stage header (title + badge) — fadeSlideUp, 0.4s
   - Beat 2: Screenshot cards — fadeSlideUp with 150ms stagger, starts 0.15s after beat 1
   - Beat 3: Annotation dots — scaleIn with back.out bounce, 100ms stagger, starts 0.3s after beat 2
   - Beat 4: Summary card — fadeSlideUp, starts 0.2s after beat 3
   This is similar to what exists but should ensure sections DON'T animate until they scroll into view (not all at once).
2. **Visual separator between stages**: Add a subtle horizontal divider between each StageSection — a thin line (1px) with a gradient from transparent → forge-border/20 → transparent. Center a small dot or diamond shape in the middle. This creates visual breathing room.
3. **Health summary entrance**: The FunnelHealthSummary should have a more dramatic entrance since it's the payoff:
   - Container scales from 0.95→1.0 while fading in (0.6s)
   - Score counter starts only when the component scrolls into view (not on mount)
   - Stat cards cascade in with 100ms stagger after the score finishes counting
4. **Blueprint CTA entrance**: The BlueprintCTA should scaleIn with the `popIn` preset (from FIX-0026) — a bouncy entrance that draws attention as the final call to action.
5. **Stage numbering**: Add a small stage number indicator (1/5, 2/5, etc.) next to each stage header to give the user a sense of progress through the results.

## Constraints
- Do NOT modify Backend or AI Engine files
- Do NOT change the stage ordering logic in ScanLayout
- ScrollTrigger `toggleActions` should be `play none none none` (play once, don't reverse)
- All animations use GSAP presets from gsap-presets.ts
- Document every animation sequence in comment blocks per CLAUDE.md rules
- Ensure the email gate blur still works (animations should play after unblur, not before)

## Verification
1. Stages reveal one at a time as user scrolls (not all at once)
2. Annotation dots bounce in with stagger after their screenshot appears
3. Visual separators visible between stages
4. Health summary score counter starts when scrolled into view
5. Stage numbers (1/5, 2/5) visible in headers
6. `npx tsc --noEmit` passes
