# FIX-0022: Blueprint funnel map — node interactivity

**Audit ref:** AUDIT-003, Phase A item 3
**Assigned to:** Frontend
**Priority:** P3 (reliability)
**Status:** pending

## Problem
BlueprintView funnel map nodes have GSAP entrance animations (fadeSlideUp with 120ms stagger) but zero user interactivity. Nodes are static divs — no hover effects, no click-to-expand, no visual feedback on interaction. The spec calls for "interactive SVG/HTML rendering" but the current implementation is purely presentational. The watermark is already present in both BlueprintView overlay AND mockup HTML, so that part is done.

## File(s) to modify
- `src/components/scan/BlueprintView.tsx` — FunnelStageNode component (around line 104-112) and its rendering

## Required change
1. **Hover state:** Add a GSAP-driven hover effect on each FunnelStageNode:
   - Subtle scale (`1 → 1.02`) and border glow (gold accent shadow) on mouseenter
   - Reverse on mouseleave
   - Use `gsap.to()` with 0.2s duration, not CSS hover
2. **Click-to-expand:** Add an expanded/collapsed state to each node:
   - Default: show stage label, health icon, and current/optimized one-liners
   - Expanded: reveal the full `improvements` bullet list and detailed descriptions
   - Animate expand with GSAP: height auto-expand + content fadeSlideUp
   - Only one node expanded at a time (accordion behavior)
3. **Active indicator:** The currently expanded node gets the gold accent border (`border-forge-accent`) instead of health-based color
4. Document all interactions in a comment block per CLAUDE.md animation rules

## Constraints
- Do NOT modify files outside Frontend-owned directories
- Do NOT change the GSAP entrance animation sequence — it already works correctly
- Do NOT modify `contracts/types.ts` — FunnelMapNode type is sufficient
- Use GSAP for all interaction animations, NOT CSS transitions (per Rule 8)
- Preserve the existing watermark overlay — do not remove or reposition it

2. Confirming GSAP is used (not CSS transitions) for hover and expand animations

4. Confirming animation comment block exists
