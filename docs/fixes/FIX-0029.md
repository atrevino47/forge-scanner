# FIX-0029: Health score visual upgrade — radial gauge + stat polish

**Audit ref:** AUDIT-003, Phase D polish
**Assigned to:** Frontend
**Priority:** P3 (reliability)
**Status:** pending

## Problem
The FunnelHealthSummary displays the health score as a plain number in a glass card. For a 19/100 score, this should feel urgent and dramatic — but it's just "19" in red text. The stat cards (stages found, missing, critical issues) are static with no hover interaction. The grid doesn't stack on mobile. The score needs visual weight to drive the user toward booking a call.

## File(s) to modify
- `src/components/scan/FunnelHealthSummary.tsx` — radial gauge, stat cards, responsive grid

## Required change
1. **Radial gauge**: Replace the plain score number with an SVG radial arc (semicircle or full circle). The arc fills proportionally (19% = small red arc, 70% = large green arc). Use GSAP to animate the arc fill from 0 to target over 1.5s (synced with the existing count-up). Color gradient: red (<30) → yellow (30-60) → green (>60) using design token colors (`forge-critical`, `forge-warning`, `forge-positive`).
2. **Score number centered in gauge**: Keep the large count-up number centered inside the radial arc. Keep JetBrains Mono font.
3. **Stat card hover**: Add GSAP hover effect on each stat card — scale 1→1.03, subtle border glow matching the stat's semantic color (green for found, yellow for missing, red for critical). 0.2s transition.
4. **Responsive grid**: Change stat card grid from fixed 3-column to `grid-cols-1 sm:grid-cols-3` so cards stack vertically on mobile.
5. **Critical issues icon fix**: The critical issues card currently uses a warning/yellow icon. Change to `AlertTriangle` or `CircleX` with `text-forge-critical` (red) to match severity.

## Constraints
- Do NOT modify Backend or AI Engine files
- Do NOT change the score calculation logic — only the visual presentation
- SVG radial gauge must be pure SVG + GSAP animation (no chart libraries)
- Keep the existing ScrollTrigger entrance animation sequence — the gauge animation should integrate with it
- Document the gauge animation in the existing comment block

## Verification
1. Score displays inside a radial arc that fills proportionally
2. Arc color matches severity (red/yellow/green)
3. Arc fill animates in sync with count-up number
4. Stat cards have GSAP hover effect
5. Grid stacks to 1-column on mobile
6. `npx tsc --noEmit` passes
