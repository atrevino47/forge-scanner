# FIX-0024b: Cal.com embed in chat — Backend stream parser

**Audit ref:** AUDIT-003, Phase A item 5
**Assigned to:** Backend
**Priority:** P2 (breaks user flow)
**Status:** pending

## Problem
The chat stream API (`/api/chat/stream/[convId]/route.ts:179-275`) has no logic to detect `[CALCOM_EMBED]` markers in the AI's response or emit `calcom_embed` SSE events. The Cal.com URL is already passed to the system prompt builder (`route.ts:156-160`) but never extracted from responses to trigger the frontend embed.

## File(s) to modify
- `src/app/api/chat/stream/[convId]/route.ts` — around line 179-275: add marker detection and SSE emission

## Required change
1. **Detect `[CALCOM_EMBED]` markers** in the streaming token buffer (same buffer mechanism as FIX-0023b — these should share the parsing infrastructure).
2. **When detected:**
   - Emit a `calcom_embed` SSE event: `data: {"type":"calcom_embed","url":"..."}\n\n`
   - The URL comes from the calcomUrl already available in the route handler (line 156-160)
   - Strip the `[CALCOM_EMBED]` text from the displayed message
3. **Emit only once per conversation** — if the AI includes the marker multiple times, only the first triggers an event. Subsequent markers are silently stripped.
4. This can share the token buffer and marker detection logic with FIX-0023b (data cards). Build a unified marker parser that handles both `[DATA_CARD:...]` and `[CALCOM_EMBED]`.

## Constraints
- Do NOT modify AI Engine files — FIX-0024a handles the system prompt
- Do NOT modify Frontend files — ChatContainer already handles `calcom_embed` events
- Do NOT modify `contracts/events.ts` — the event type is correct
- The emitted event must match the `calcom_embed` shape in `contracts/events.ts:102`
- Must coordinate with FIX-0023b to share the marker parsing infrastructure

## Verification
The Orchestrator will verify by:
1. Reading the stream route and confirming `[CALCOM_EMBED]` detection exists
2. Confirming `calcom_embed` SSE event is emitted with correct shape
3. Confirming marker text is stripped from displayed message
4. Confirming only-once-per-conversation guard exists
5. `npx tsc --noEmit` passes

## Cross-agent context
FIX-0024a (AI Engine) defines the `[CALCOM_EMBED]` marker protocol. **FIX-0024a runs first.** FIX-0023b (Backend) implements the shared marker parser — this ticket and FIX-0023b should be implemented together as they share the same parsing infrastructure in the stream route.
