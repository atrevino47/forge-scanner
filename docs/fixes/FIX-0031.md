# FIX-0031: Error states + loading skeletons

**Audit ref:** AUDIT-003, Phase D.7 + D.8
**Assigned to:** Frontend
**Priority:** P2 (breaks user flow)
**Status:** pending

## Problem
1. BlueprintCTA has a `// TODO: error toast` comment — when blueprint generation fails, the button silently reappears with no feedback. Users don't know something went wrong.
2. SkeletonLoader is a single generic `h-64` box used everywhere. No structural variants that hint at the content being loaded (text lines, image placeholder, etc.).
3. No loading timeout — if blueprint API hangs, the "Generating..." state spins forever.
4. Screenshot image load failures show nothing (blank space).

## File(s) to modify
- `src/components/scan/BlueprintCTA.tsx` — error state UI, timeout, loading spinner
- `src/components/scan/SkeletonLoader.tsx` — multiple skeleton variants
- `src/components/scan/StageSection.tsx` — use structured skeleton variant

## Required change
1. **BlueprintCTA error state**: When the API call fails, show an error state inside the CTA card:
   - Red-tinted glass card with `AlertTriangle` icon
   - "[COPY: blueprint generation failed]" message
   - Gold "Try Again" button that resets error and re-triggers generation
   - Add a 45-second timeout: if the API hasn't responded, abort the fetch and show the error state
   - While generating: show a spinner icon (Loader2 with spin) next to the button text
2. **SkeletonLoader variants**: Create named skeleton variants:
   - `screenshot`: Matches aspect-video ratio with a small device badge placeholder in corner
   - `text-block`: 3-4 lines of varying width (100%, 85%, 70%, 40%) to hint at paragraph text
   - `stage-summary`: Full-width card with 2 text lines + score badge placeholder
   - Keep the shimmer animation overlay on all variants
3. **StageSection skeleton**: When a stage is in 'analyzing' status with no screenshots yet, show the `screenshot` skeleton variant (not the generic box). When summary is pending, show `stage-summary` variant.

## Constraints
- Do NOT modify Backend or AI Engine files
- Do NOT change the blueprint API contract or response format
- Skeleton variants must all use the existing shimmer animation (fixed in FIX-0026)
- Error state uses design token colors (forge-critical for error accent)

## Verification
1. Blueprint generation failure shows error card with "Try Again" button
2. Clicking "Try Again" re-triggers the API call
3. 45-second timeout fires if API hangs
4. SkeletonLoader has at least 3 named variants
5. StageSection uses structured skeletons
6. `npx tsc --noEmit` passes
