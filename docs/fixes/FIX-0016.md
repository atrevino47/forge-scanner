# FIX-0016: Add exit detection and trigger follow-up sequence

**Audit ref:** AUDIT-001, MISSING items 7-8
**Assigned to:** Frontend
**Priority:** P4 (wiring)
**Status:** pending

## Problem

The spec (§8) requires exit detection via `visibilitychange` + `beforeunload` to trigger follow-up sequences when a user leaves without booking. The Backend API at `/api/followup/trigger` already exists but nothing calls it. No browser-side exit detection exists.

## File(s) to modify

- `src/components/scan/ScanLayout.tsx` — add a useEffect for exit detection

## Required change

Add a useEffect in the ScanLayout component that:
1. Listens for `visibilitychange` (tab switch/close) and `beforeunload` (page close/navigate away)
2. When the user leaves AND has an email captured AND hasn't booked, fires a POST to `/api/followup/trigger`
3. Uses `navigator.sendBeacon()` for reliability (works even during page unload)
4. Only fires once per session (use a ref to track)

Add this useEffect after the existing chat timer effect (~line 362):

```typescript
// ── Exit detection — trigger follow-up if user leaves without booking ──
const exitTriggeredRef = useRef(false);

useEffect(() => {
  if (!state.emailCaptured || !state.leadId || hasBooked) return;

  const triggerExit = (reason: 'exit_intent' | 'abandoned_scan') => {
    if (exitTriggeredRef.current || hasBooked) return;
    exitTriggeredRef.current = true;

    const payload = JSON.stringify({
      scanId,
      leadId: state.leadId,
      reason,
    });

    // sendBeacon is reliable during page unload
    navigator.sendBeacon('/api/followup/trigger', new Blob([payload], { type: 'application/json' }));
  };

  const handleVisibilityChange = () => {
    if (document.visibilityState === 'hidden' && state.completedSummary) {
      triggerExit('exit_intent');
    }
  };

  const handleBeforeUnload = () => {
    if (state.completedSummary) {
      triggerExit('exit_intent');
    } else {
      triggerExit('abandoned_scan');
    }
  };

  document.addEventListener('visibilitychange', handleVisibilityChange);
  window.addEventListener('beforeunload', handleBeforeUnload);

  return () => {
    document.removeEventListener('visibilitychange', handleVisibilityChange);
    window.removeEventListener('beforeunload', handleBeforeUnload);
  };
}, [scanId, state.emailCaptured, state.leadId, state.completedSummary, hasBooked]);
```

## Constraints

- Do NOT modify the followup/trigger API route
- Do NOT modify any other component
- Use `navigator.sendBeacon` — not `fetch` — because fetch is unreliable during page unload
- Only trigger once per session (ref guard)
- Only trigger if email was captured (no point following up without contact info)
- Do NOT trigger if user already booked
