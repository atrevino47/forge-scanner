# FIX-0026: CSS & animation foundation fixes

**Audit ref:** AUDIT-003, Phase D prerequisites
**Assigned to:** Frontend
**Priority:** P1 (blocks pipeline)
**Status:** pending

## Problem
Three broken/violating animation foundations that affect the entire results page:
1. `@keyframes shimmer` in globals.css is missing its `from` state — skeleton loaders render static
2. `@keyframes blink` is referenced in ChatMessage.tsx but never defined — streaming cursor is invisible
3. AnnotationMarker.tsx uses Tailwind `animate-pulse` which violates CLAUDE.md Rule 8 (GSAP only) and runs indefinitely (CPU waste)

## File(s) to modify
- `src/app/globals.css` — add/fix keyframes
- `src/components/scan/AnnotationMarker.tsx` — replace Tailwind pulse with GSAP
- `src/lib/gsap-presets.ts` — add `scaleOut()` and `popIn()` preset factories for use across Phase D

## Required change
1. **globals.css**: Fix `@keyframes shimmer` to include `from { transform: translateX(-100%); }`. Add `@keyframes blink { 0%, 100% { opacity: 1; } 50% { opacity: 0; } }`.
2. **AnnotationMarker.tsx**: Remove `animate-pulse` class. Add a GSAP `gsap.to()` scale pulse (1.0 → 1.1 → 1.0) that runs twice on mount then stops. Add `cursor-pointer` class. Add `aria-label` with annotation title for accessibility.
3. **gsap-presets.ts**: Add `scaleOut()` preset (scale 1→0.9, opacity 1→0, 0.25s) for dismiss animations. Add `popIn()` preset (scale 0.8→1 with back.out(1.7) ease, 0.4s) for attention elements.

## Constraints
- Do NOT modify Backend or AI Engine files
- Do NOT change the visual appearance of existing animations — only fix broken ones
- New GSAP presets must follow the existing factory pattern (return `{ from, vars }`)

## Verification
1. SkeletonLoader shimmer animates left-to-right continuously
2. ChatMessage streaming cursor blinks
3. AnnotationMarker pulsing uses GSAP, not Tailwind
4. `npx tsc --noEmit` passes
