# FIX-0006: Duplicate screenshot keys from SSE + initial data race

**Audit ref:** AUDIT-002, runtime testing
**Assigned to:** Frontend
**Priority:** P2 (breaks user flow — React key warnings, potential duplicate renders)
**Status:** pending

## Problem

ScanLayout has two data sources that both add screenshots to state: (1) the initial fetch from `/api/scan/results/{scanId}` via `INITIAL_DATA` action, and (2) the SSE stream emitting `screenshot_captured` events. The `screenshot_captured` handler (line 151-170) always appends without checking if the screenshot ID already exists. When both fire, the same screenshot appears twice with the same `id`, causing React's "two children with the same key" error.

## File(s) to modify

- `src/components/scan/ScanLayout.tsx` — lines 151-170: the `screenshot_captured` case in `handleSSEEvent`

## Required change

Before appending a new screenshot, check if one with the same `id` already exists in `state.screenshots`. If it does, skip the append.

**Before:**
```typescript
case 'screenshot_captured':
  return {
    ...state,
    screenshots: [
      ...state.screenshots,
      {
        id: event.screenshotId,
        stage: event.stage,
        source: event.source,
        thumbnailUrl: event.thumbnailUrl,
        viewport: event.viewport,
        annotations: [],
        isNew: true,
      },
    ],
```

**After:**
```typescript
case 'screenshot_captured': {
  if (state.screenshots.some((s) => s.id === event.screenshotId)) {
    return state;
  }
  return {
    ...state,
    screenshots: [
      ...state.screenshots,
      {
        id: event.screenshotId,
        stage: event.stage,
        source: event.source,
        thumbnailUrl: event.thumbnailUrl,
        viewport: event.viewport,
        annotations: [],
        isNew: true,
      },
    ],
```

Note: the case now uses a block (`{ }`) because of the early return. Make sure the closing brace and trailing comma/semicolons are correct.

## Constraints

- Do NOT modify the `INITIAL_DATA` handler or the SSE connection logic
- Do NOT modify the `annotation_ready` handler (it correctly uses `.map()` to update in place)
- Only add the dedup guard to `screenshot_captured`
