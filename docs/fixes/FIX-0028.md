# FIX-0028: Annotation dot ↔ summary list hover interaction

**Audit ref:** AUDIT-003, Phase D polish
**Assigned to:** Frontend
**Priority:** P3 (reliability)
**Status:** pending

## Problem
Annotation dots and the summary list below each screenshot are visually disconnected. Hovering a summary item gives no visual link to its corresponding dot on the image, and vice versa. The popover also closes instantly with no exit animation. These gaps make the annotation system feel static instead of interactive and premium.

## File(s) to modify
- `src/components/scan/ScreenshotCard.tsx` — add hover state coordination between dots and list
- `src/components/scan/AnnotationMarker.tsx` — add highlighted state + hover scale
- `src/components/scan/AnnotationPopover.tsx` — add exit animation, improve positioning

## Required change
1. **Bidirectional hover**: Add a `hoveredAnnotationIndex` state to ScreenshotCard. When user hovers a summary list item, set the index → the corresponding AnnotationMarker on the image gets a highlight ring (gold glow via GSAP). When user hovers a dot, the corresponding summary list item gets a highlight background. Use GSAP for both transitions (0.2s).
2. **Marker hover scale**: When hovering any inactive AnnotationMarker, scale it to 1.15 with GSAP (0.15s). This provides clickability feedback beyond just cursor-pointer.
3. **Popover exit animation**: When the active annotation changes or is dismissed, animate the popover out with `scaleOut()` preset (scale 1→0.9, opacity 1→0, 0.2s) before unmounting. Currently it vanishes instantly.
4. **Summary list scroll-into-view**: When a dot is clicked and the summary list is below the viewport fold, smoothly scroll the corresponding list item into view.

## Constraints
- Do NOT modify Backend or AI Engine files
- Do NOT change the annotation data format or positioning percentages
- Hover highlighting must use GSAP (not CSS :hover pseudo-class) per Rule 8
- The existing click-to-open-popover behavior must still work

## Verification

2. Hovering dot #2 on the image highlights summary item #2
3. Popover animates out (not instant vanish)
