# FIX-0004b: Remove empty email from SocialConfirmation request

**Audit ref:** AUDIT-001, BUG-3
**Assigned to:** Frontend
**Priority:** P2 (breaks user flow)
**Status:** pending

## Problem

The SocialConfirmation component sends `email: ''` in its capture-info request body (line 58). This empty string fails the Backend's Zod email validation. After the Backend fix (FIX-0004a), email is optional — but the Frontend should stop sending it entirely when it's not applicable.

## File(s) to modify

- `src/components/scan/SocialConfirmation.tsx` — lines 52-64: the `handleConfirm` function's fetch body

## Required change

Remove the `email: ''` field from the request body. Only send `scanId`, `leadId`, and `socialConfirmation`.

**Before:**
```typescript
await fetch('/api/scan/capture-info', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    scanId,
    leadId: leadId ?? '',
    email: '',
    socialConfirmation: {
      platform,
      confirmedHandle: selected,
    },
  }),
});
```

**After:**
```typescript
await fetch('/api/scan/capture-info', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    scanId,
    leadId: leadId ?? '',
    socialConfirmation: {
      platform,
      confirmedHandle: selected,
    },
  }),
});
```

## Constraints

- Do NOT modify any other component or file
- Do NOT add email capture to this component — that's CapturePrompt's job
- The `leadId ?? ''` pattern can remain (it's handled separately by RUNTIME-4 / FIX-0005)

## Cross-agent context

This is sub-ticket **b** (Frontend). Apply AFTER FIX-0004a (Backend).

FIX-0004a updates the Backend Zod schema to make `email` optional when `socialConfirmation` is present. This ticket removes the empty email from the Frontend request so the two sides align.

## Verification

The Orchestrator will verify by:
1. `npx tsc --noEmit` — zero errors
2. Confirm `email` is not in the `JSON.stringify` body of `handleConfirm`
3. Confirm the request still includes `scanId`, `leadId`, and `socialConfirmation`
