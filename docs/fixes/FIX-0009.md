# FIX-0009: Screenshots need scrollable container and annotation dots need labels

**Audit ref:** AUDIT-002, E2E browser test
**Assigned to:** Frontend
**Priority:** P2 (breaks user flow — results page is unusable with giant images)
**Status:** pending

## Problem

Two related display issues on the scan results page:

1. **Screenshots are too tall to view:** Full-page desktop screenshots are thousands of pixels tall. The `<img>` renders at `w-full` with no height constraint, forcing users to scroll endlessly through each stage section. Screenshots need a fixed-height scrollable container.

2. **Annotation dots are not intuitive:** The `AnnotationMarker` renders a 24px colored circle with a tiny inner dot. There's no number, no label, no visual hint that the dots are interactive. Users don't know to click them. The dots need numbered badges and the annotation title should be visible without requiring a click.

## File(s) to modify

- `src/components/scan/ScreenshotCard.tsx` — screenshot container and annotation list
- `src/components/scan/AnnotationMarker.tsx` — marker appearance

## Required change

### Change 1: Scrollable screenshot container (ScreenshotCard.tsx)

Wrap the screenshot image in a fixed-height container with vertical scrolling.

**Before (line 54):**
```tsx
<div className="relative">
  <img
    src={screenshot.thumbnailUrl}
    alt={`${screenshot.source} screenshot (${screenshot.viewport})`}
    className="w-full"
    loading="lazy"
  />
  {/* Annotation dots overlaid */}
```

**After:**
```tsx
<div className="relative max-h-[600px] overflow-y-auto">
  <div className="relative">
    <img
      src={screenshot.thumbnailUrl}
      alt={`${screenshot.source} screenshot (${screenshot.viewport})`}
      className="w-full"
      loading="lazy"
    />
    {/* Annotation dots overlaid */}
```

Add the corresponding closing `</div>` for the new wrapper. The annotation dots and popover must stay INSIDE the inner `relative` div (so their absolute positioning still works relative to the image). The outer div with `max-h-[600px] overflow-y-auto` creates the scroll container.

Also add an annotation summary list BELOW the screenshot scroll container (but still inside the card) so users can see all findings without hunting for dots:

```tsx
{/* Annotation list below screenshot */}
{screenshot.annotations.length > 0 && (
  <div className="border-t border-forge-border px-4 py-3 space-y-2">
    {screenshot.annotations.map((annotation, index) => (
      <button
        key={annotation.id}
        onClick={() => handleMarkerClick(annotation)}
        className="flex w-full items-start gap-3 rounded-lg px-2 py-1.5 text-left transition-colors duration-200 hover:bg-forge-surface"
      >
        <span className={cn(
          'mt-0.5 flex h-5 w-5 shrink-0 items-center justify-center rounded-full text-[10px] font-bold text-forge-base',
          TYPE_BG[annotation.type]
        )}>
          {index + 1}
        </span>
        <div>
          <p className="font-body text-sm font-medium text-forge-text">{annotation.title}</p>
          <p className="font-body text-xs text-forge-text-muted line-clamp-1">{annotation.detail}</p>
        </div>
      </button>
    ))}
  </div>
)}
```

This requires importing `TYPE_BG` from `AnnotationMarker.tsx` or defining it locally, and importing `cn` from `@/lib/utils`. Both are already available patterns in the codebase.

### Change 2: Numbered annotation markers (AnnotationMarker.tsx)

Add an `index` prop and display the number inside the dot. Add a pulse animation for unclicked dots.

**Updated component interface:**
```tsx
interface AnnotationMarkerProps {
  annotation: Annotation;
  index: number; // NEW: 0-based index for display as 1-based number
  isActive: boolean;
  onClick: () => void;
}
```

**Updated marker render:**
```tsx
<button
  data-annotation-dot
  onClick={onClick}
  className={cn(
    'absolute z-10 flex h-7 w-7 items-center justify-center rounded-full border-2 border-forge-base text-[10px] font-bold text-forge-base',
    TYPE_BG[annotation.type],
    isActive && 'ring-2 ring-forge-text/30 ring-offset-1 ring-offset-forge-base',
    !isActive && 'animate-[pulse_2s_ease-in-out_infinite]',
  )}
  style={{
    left: `${annotation.position.x}%`,
    top: `${annotation.position.y}%`,
    transform: 'translate(-50%, -50%)',
  }}
  title={annotation.title}
>
  {index + 1}
</button>
```

**Update ScreenshotCard.tsx** to pass `index` to each AnnotationMarker:
```tsx
{screenshot.annotations.map((annotation, index) => (
  <AnnotationMarker
    key={annotation.id}
    annotation={annotation}
    index={index}
    isActive={activeAnnotation?.id === annotation.id}
    onClick={() => handleMarkerClick(annotation)}
  />
))}
```

Also export `TYPE_BG` from AnnotationMarker.tsx so ScreenshotCard can use it for the annotation list.
